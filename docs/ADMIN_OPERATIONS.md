# Admin Operations Guide

## Daily responsibilities

- Verify new student accounts can log in
- Confirm batches, enrollments, and announcements are up to date
- Review unresolved doubts
- Confirm live class schedules are correct
- Check that recently suspended users cannot log back in

## Before a new batch starts

- create or verify the course
- create the batch with correct dates and capacity
- enroll the student list
- confirm lectures, notes, and tests are visible
- publish the announcement for batch onboarding
- verify one student test login works on a real device

## Creating a teacher account

- use the `Add Teacher` action in the admin teacher screen
- enter teacher name, teacher email, and a temporary password
- confirm the success message appears
- ask the teacher to check their email for the login credentials
- ask the teacher to reset the password after first login if needed

## If a student loses access unexpectedly

- check whether the account is active
- verify the student is still enrolled in the expected batch
- confirm the course is published
- ask the student to sign out and sign in again
- if offline downloads stopped working, ask them to re-download after access is restored

## If a teacher cannot manage content

- confirm the teacher account is active
- confirm the teacher owns the course/batch they are trying to access
- verify the content belongs to that course structure

## Important note

Offline downloads are intentionally basic in this version. If a student is unenrolled or loses institute access, offline content may stop working and need to be removed or re-downloaded.
